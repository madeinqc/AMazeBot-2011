Requirements

This program requires Java version 1.6. If you will be developing your own bot, this entire folder is a NetBeans project (it was tested on version 6.9). Other IDEs could be used, but you'd be on your own in terms of getting it working.

Bot Icon and Name

In the "./src/bots/SampelBot/" folder you will see a file called "icon.png". This is the image that will identify your bot, and you should replace it with one of your choosing. The image will be resized to be 80 by 80 pixels when loaded, but to avoid distortion we recommend that you prepare the image to the appropriate size in an image editing program (such as GIMP). Choose a nice image—it will be shown to the public during the competition!

Also note that in the source code you can designate a nickname for your bot--see the getName() method. The maximum length of this name is approximately 15 'x' characters (we can't be exact, since it depends on the font and characters used). If you are unsure whether your desired name will fit, send an email to me (Aravin) with the desired name and I'll check for you. Don't leave this to the last minute!

Sharing Bots

If you want to let a friend run your bot (but not see the source code) you can Clean and Build your project, and send them the subfolder of "./bots/" that corresponds to your bot. Your friend can then drop that subfolder into their "./bots/" directory and run your bot from the Developer Module just as they would run their own.

Batch Mode

In addition to the GUI, the program can be run in a batch mode from the command line. This is useful for testing your bot on a large number of randomly generated mazes.

Invoke batch mode thusly: 

   java -jar AMazeBot2010.jar batch <botname> [options]

Where

Where <botname> is the package name of your bot (e.g. "SampleBot") and [options] can be any combination of:

  • -i <iterations> — to specify number of mazes to run; <iterations> is a number; default is 50,000.
  • -s <sequence> — to specify the sequence; <sequence> is a non-negative integer; default is random.
  • -m <maze type> — to select the maze type; <maze type> is one of CB, GR, NS, RG, PR; default is Automatic.
  • -d — to produce detailed action counts in CSV output.

Note that the botname is case-sensitive; options are not. You can stop a batch at any time using CTRL-C and still get the results up to that point.

The results of batch testing are stored in the "./logs/" folder. There are 2 files: "<botname>_batch.csv" and "<botname>_batch.txt". The CSV file contains information about each maze run and can be loaded directly into a spreadsheet for analysis. The TXT file contains the same summary information that is shown on the console after a batch. These files will be overwritten with each new batch.

There is a single parameter in the INI file that affects batch mode—the "batch_timeout" key under the "BATCH" section. It is the maximum number of seconds to allow a bot to run before considering it caught in an infinite loop, and terminating it. In the GUI you have the "Abort Test" button to interrupt the bot at any time, but of course in batch mode the system itself has to prevent a bot from running forever. The default value for this timeout is 5 seconds; it can be as low as 1s and high as 60s. If your bot is just slow in processing (for example if you're writing a map to a file every iteration of your main loop) you can increase this value.
